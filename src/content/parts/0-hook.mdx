# Hook

A prompt is just text the model continues. When you press enter, the system does
not retrieve a finished answer from a database. It repeatedly asks one question:
given the context so far, which <Token text="token" /> should come next?

<PromptToken client:visible />

The selected token is appended to the context, so the next step sees a slightly
longer prompt. That simple loop is why the rest of the descent matters. The model
creates the distribution, weights store what it learned, serving software keeps
the loop efficient, and hardware supplies the matrix math and memory bandwidth.

<DeeperBlock summary="Autoregressive loop">
  Autoregressive generation means `context -> logits -> sample -> append -> repeat`.
  The model is not planning the whole answer in one pass. It is repeatedly scoring
  a vocabulary-sized list of next-token candidates under the current context.
</DeeperBlock>
