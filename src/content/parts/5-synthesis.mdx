# Put it together

The opening prompt is now legible as a full stack. A <Token text="token" /> starts
as text, becomes IDs and vectors, moves through attention and feed-forward layers,
uses weights stored in finite precision, enters a serving loop with KV cache, and
lands on tensor cores and memory bandwidth before one visible next token appears.

<FullStackReplay client:visible />

## One concrete pass

Take the running prompt: `The GPU keeps KV cache ready for`. Tokenization turns it
into IDs. Embeddings and RoPE give those IDs content and position. Attention lets
the position after `for` read earlier tokens like `GPU`, `KV`, and `cache`.
Feed-forward layers transform the stream, and the final projection produces logits
over the vocabulary. Sampling may choose <Token text="decode" />.

The system then appends that token and repeats. The serving engine keeps the
prompt's keys and values in cache so the next decode step does not recompute the
whole prompt. The hardware still has to read weights, move cached values, and run
the tiled matrix math for each new token.

## Configure the path

The same model can feel very different depending on quantization, context length,
and hardware. A small quantized model may fit comfortably on a workstation GPU.
A larger FP16 model may need a datacenter card just to load the weights, before
the KV cache and runtime overhead are counted.

For example, a 7B dense model at Q4 is roughly 3.5 GB of raw weight bytes. With a
4k context, GQA-style KV cache, and runtime overhead, the first-order estimate is
still comfortably inside a 24 GB workstation GPU. A 70B model at FP16 is a
different class of problem: the weights alone are far beyond that card. The same
architecture idea is running, but the capacity and bandwidth terms dominate the
deployment decision.

The practical mental model is compact: the model decides the distribution, the
sampler chooses from it, the serving engine keeps the loop full, and the hardware
sets the capacity and bandwidth envelope. Most inference debugging is figuring
out which one of those layers is actually limiting the next token.
