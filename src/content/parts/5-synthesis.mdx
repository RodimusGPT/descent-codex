# Put it together

The opening prompt is now legible as a full stack. A <Token text="token" /> starts
as text, becomes IDs and vectors, moves through attention and feed-forward layers,
uses weights stored in finite precision, enters a serving loop with KV cache, and
lands on tensor cores and memory bandwidth before one visible next token appears.

## Configure the path

The same model can feel very different depending on quantization, context length,
and hardware. A small quantized model may fit comfortably on a workstation GPU.
A larger FP16 model may need a datacenter card just to load the weights, before
the KV cache and runtime overhead are counted.
